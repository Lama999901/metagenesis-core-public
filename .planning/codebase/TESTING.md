## Test Categories

### 1. Claim Verification Tests
Each claim has its own test file with at minimum:
- **Pass test:** Valid inputs produce PASS result
- **Fail test:** Invalid/out-of-tolerance inputs produce FAIL
- **Determinism test:** Same seed → identical `trace_root_hash`

Example pattern (`tests/materials/test_mtr1_youngs_modulus.py`):
```python
def test_a_mtr1_pass():
    result = run_mtr1(E_measured=70.0, E_reference=70.0)
    assert result["execution_trace"][-1]["output"]["pass"] is True

def test_b_mtr1_fail():
    result = run_mtr1(E_measured=80.0, E_reference=70.0)
    assert result["execution_trace"][-1]["output"]["pass"] is False

def test_c_mtr1_determinism():
    r1 = run_mtr1(...)
    r2 = run_mtr1(...)
    assert r1["trace_root_hash"] == r2["trace_root_hash"]
```

## Mocking Strategy

**No mocking.** All tests run against real implementations:
- No database dependencies (stdlib only)
- No network calls
- No external services
- Deterministic by design (seeded RNG where applicable)

This is intentional — the protocol's value proposition requires real execution to produce real hashes.

## Key Testing Patterns

### Step Chain Verification
Every claim test verifies the 4-step chain structure:
```python
trace = result["execution_trace"]
assert len(trace) == 4
assert trace[0]["name"] == "init_params"
assert trace[-1]["name"] == "threshold_check"
assert result["trace_root_hash"] == trace[-1]["hash"]
```

### Evidence Normalization
Tests that compare artifacts strip temporal/path-dependent fields:
```python
def _normalize_evidence(ev):
    # Remove timestamps, absolute paths, UUIDs
    # Keep only deterministic content
```

### Semantic Verification
Tests import `_verify_semantic()` directly from `scripts/mg.py` — no subprocess calls needed for Layer 2 tests.
