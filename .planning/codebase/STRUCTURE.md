## File Naming Conventions

### Claim modules: `backend/progress/`
- Pattern: `<claim_id_lowercase>.py` (e.g., `mtr1_calibration.py`, `mlbench2_regression_certificate.py`)
- Each file contains one claim function returning a standard result dict

### Test files: `tests/<domain>/`
- Pattern: `test_<claim_id_or_feature>.py`
- Domain directories: `steward`, `materials`, `ml`, `digital_twin`, `data`, `systems`, `cli`, `progress`
- Adversarial tests: `test_cert0N_<description>.py` (cert01 through cert08)

### Scripts: `scripts/`
- Core tools: `mg.py`, `mg_sign.py`
- Governance: `steward_*.py`
- Policy: `mg_policy_gate*.py`

### Reports: `reports/`
- Run artifacts: `job_job_<uuid>_trace_trace_<uuid>_success.json`
- Ledger snapshots: `trace_trace_<uuid>.jsonl`

## Key Locations

| What | Where |
|------|-------|
| Add new claim | `backend/progress/<claim>.py` |
| Register claim dispatch | `backend/progress/runner.py` |
| Add tests | `tests/<domain>/test_<claim>.py` |
| Test fixtures (CSV) | `tests/fixtures/` |
| Claim registry | `reports/scientific_claim_index.md` |
| Update counters | `index.html` (11 places), `README.md`, `system_manifest.json` |
| Policy gate allowlist | `scripts/mg_policy_gate_policy.json` |
| Known limitations | `reports/known_faults.yaml` |

---

## Where to Place New Code

- **New claim:** `backend/progress/<claim_id>.py` + follow 6-step process in CLAUDE.md
- **New verification layer:** `scripts/mg.py` (modify carefully — semi-sealed)
- **New governance script:** `scripts/steward_<name>.py`
- **New test domain:** `tests/<domain>/` with `__init__.py`
- **New documentation:** `docs/<TOPIC>.md`
- **New demo:** `demos/<demo_name>/`
