**PILOT-05 — CLI with argparse:**
- `--process FILE` — process a CSV file of submissions
- `--status` — show current queue state from reports/pilot_queue.json
- `--mark-sent EMAIL` — mark a processed entry as sent
- `--help` — auto from argparse
- No args → print help

**PILOT-01 — CSV reading and domain detection:**
- `detect_domain(description: str) -> str` function
- Keyword map (case-insensitive matching on description text):
  - ml/ai/model/accuracy/benchmark/neural/deep learning → "ml"
  - drug/pharma/admet/fda/clinical/compound → "pharma"
  - material/steel/aluminum/titanium/copper/modulus/conductivity → "materials"
  - var/risk/basel/portfolio/finance → "finance"
  - fem/sensor/twin/iot/displacement/calibration → "digital_twin"
- Default to "ml" if no keywords match (per user decision), set flagged_for_review=true in queue entry
- `read_submissions(csv_path: str) -> list[dict]` using csv.DictReader
- Expected columns: name, email, company, domain_description, date
- Skip rows with missing name or email (log warning to stderr)

**PILOT-04 — Queue state management:**
- `load_queue() -> dict` reads reports/pilot_queue.json (creates empty {"entries": []} if missing)
- `save_queue(queue: dict)` writes reports/pilot_queue.json with indent=2, sort_keys=True
- Queue entry schema: {"name", "email", "company", "domain_description", "date", "domain_detected", "status", "processed_at", "bundle_path", "draft_path", "flagged_for_review"}
- Status values: "pending" → "processed" → "sent"
- `--mark-sent EMAIL` updates status from "processed" to "sent" with sent_at timestamp

**PILOT-02 — Bundle generation:**
- For each submission, call mg_client.run_claim(domain) with default params
- Call mg_client.create_bundle(result, output_dir=str(REPO_ROOT / "reports" / "pilot_bundles" / f"{name}_{date}"))
- Call mg_client.verify_bundle(bundle_dir) and capture passed status
- Store bundle_path in queue entry

**PILOT-03 — Email draft generation:**
- `generate_draft(entry: dict, claim_result: dict, bundle_dir: Path, verified: bool) -> Path`
- Create reports/pilot_drafts/ directory (parents=True, exist_ok=True per BUG 5)
- Draft filename: reports/pilot_drafts/{name}_{date}.txt (sanitize name: replace spaces with underscores, lowercase)
- Draft content (plain text):
  ```
  Subject: MetaGenesis Core -- Verification Bundle Ready

  Hi {name},

  Your MetaGenesis Core pilot verification is complete.

  Company: {company}
  Domain: {domain_detected}
  Result: PASS
  Bundle: {bundle_path}

  Your verified bundle proves that the computation produced exactly this
  result, at exactly this time, in exactly this way. It is independently
  auditable offline -- no trust required.

  Next step: Secure your full MetaGenesis Core license for $299:
  https://buy.stripe.com/14AcN57qH19R1qN3QQ6Na00

  Questions? Reply to this email.

  Best,
  Yehor Bazhynov
  MetaGenesis Core
  yehor@metagenesis-core.dev
  ```
- If verification FAIL, adjust Result line and add note about manual review needed
- Store draft_path in queue entry

**process_submissions(csv_path: str) main flow:**
1. Read CSV with read_submissions()
2. Load queue
3. For each row: detect domain, run claim, create bundle, verify, generate draft, add queue entry with status="processed"
4. Save queue
5. Print summary: N submissions processed, domains detected, all PASS/any FAIL

**--status flag:**
- Print table: name | email | domain | status | processed_at
- Print counts: N pending, N processed, N sent
