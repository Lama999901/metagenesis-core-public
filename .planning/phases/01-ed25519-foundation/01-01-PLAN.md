**Constants (RFC 8032 Section 5.1):**
```python
P = 2**255 - 19
L = 2**252 + 27742317777372353535851937790883648493
D = -121665 * pow(121666, P - 2, P) % P
I = pow(2, (P - 1) // 4, P)
KEY_VERSION = "ed25519-v1"
```

**Field arithmetic:**
- `inv(x)` -- modular inverse via Fermat: `pow(x, P - 2, P)`

**Point operations (extended coordinates: x, y, z, t):**
- `point_add(P, Q)` -- Edwards curve addition in extended coordinates
- `scalar_mult(s, P)` -- double-and-add scalar multiplication
- `point_encode(P)` -- encode point as 32 bytes (y-coordinate + x sign bit in MSB of byte 31)
- `point_decode(s)` -- decode 32 bytes to point, extract sign bit from byte 31 MSB

**Square root recovery (CRITICAL -- see Pitfall 3 in RESEARCH.md):**
```python
def recover_x(y, sign):
    u = (y * y - 1) % P
    v = (D * y * y + 1) % P
    x = pow(u * v, (P + 3) // 8, P) * u % P  # WRONG -- see correction below
    # Correct formula: x = pow(u * v**3, (P + 3) // 8, P) * u * v % P ...
    # Actually use: x2 = u * pow(v, P-2, P) % P, then x = pow(x2, (P+3)//8, P)
    # Best approach from RFC:
    #   x = pow(u * v**3 * pow(u * v**7, (P-5)//8, P), 1, P)  -- Bernstein's formula
    # OR simpler:
    #   candidate = pow(u * pow(v, P-2, P), (P+3)//8, P)
    #   if (v * candidate * candidate - u) % P != 0:
    #       candidate = candidate * I % P
    #   if (v * candidate * candidate - u) % P != 0:
    #       raise ValueError("not on curve")
    #   if candidate % 2 != sign:
    #       candidate = P - candidate
    #   return candidate
```
USE THE RESEARCH.md Pitfall 3 approach: compute `u/v` candidate, check, apply sqrt(-1) correction if needed.

h = bytearray(hashlib.sha512(private_seed).digest())

BY = 4 * pow(5, P - 2, P) % P

**Public API:**
- `generate_keypair() -> tuple[bytes, bytes]` -- returns (private_seed_32bytes, public_key_32bytes)
- `sign(private_seed: bytes, message: bytes) -> bytes` -- returns 64-byte signature
- `verify(public_key: bytes, message: bytes, signature: bytes) -> bool` -- constant-time final comparison using `hmac.compare_digest` on encoded points

**Verification equation (RFC 8032 Section 5.1.7):**
