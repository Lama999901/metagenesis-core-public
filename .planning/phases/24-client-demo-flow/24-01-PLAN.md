Create `scripts/mg_demo.py` (per DEMO-01, DEMO-02, DEMO-03) with the following behavior:

**CLI interface:**
- `python scripts/mg_demo.py` (no args) -- prints numbered domain menu, user picks by number
- `python scripts/mg_demo.py --domain materials` -- skips menu, goes straight to that domain
- `python scripts/mg_demo.py --all` -- runs demo for every domain sequentially
- `--output-dir` optional, defaults to `demos/receipts/`

**Domain mapping (from index.json):**
Build a DOMAIN_DISPLAY dict mapping friendly names to index.json domain values:
```python
DOMAIN_DISPLAY = {
    "materials": {"label": "Materials Science", "domains": ["materials"]},
    "physics": {"label": "Fundamental Physics", "domains": ["physics"]},
    "ml": {"label": "Machine Learning", "domains": ["ml"]},
    "pharma": {"label": "Pharmaceutical (ADMET)", "domains": ["pharma"]},
    "finance": {"label": "Financial Risk (Basel III)", "domains": ["finance"]},
    "digital_twin": {"label": "Digital Twin", "domains": ["digital_twin"]},
    "systems": {"label": "Systems Engineering", "domains": ["systems"]},
    "agent": {"label": "Agent Monitoring", "domains": ["agent"]},
}
```

**Demo flow (for each domain):**
1. Load `proof_library/index.json`, filter entries where `domain == chosen_domain` and `is_synthetic == False`
2. For each matching entry:
   a. Locate bundle ZIP at `entry["bundle_path"]` (resolve relative to REPO_ROOT)
   b. Extract ZIP to a temp directory (use `tempfile.mkdtemp` + `zipfile.ZipFile`)
   c. Call `verify_bundle(extracted_dir)` from `scripts.mg_client`
   d. Print 5-layer verification results with PASS/FAIL icons (reuse `_print_layer_results` pattern from mg_client.py)
   e. Load `evidence.json` from extracted bundle, call `format_receipt(claim_result)` from `scripts.mg_receipt`
   f. Print receipt to stdout
   g. Clean up temp directory
3. After all claims in domain verified, write a combined domain receipt to `demos/receipts/{domain}_receipt.txt` containing:
   - Section: "METAGENESIS CORE -- DOMAIN DEMO RECEIPT"
   - Section: "Summary" -- domain name, number of claims verified, all PASS/FAIL
   - Section: "Claims Verified" -- for each claim: claim ID, one-sentence plain-language description (from `CLAIM_DESCRIPTIONS` in mg_receipt.py), result summary, PASS/FAIL
   - Section: "Verification Result" -- "All N claims verified through 5 independent cryptographic layers"
   - Section: "How to Reproduce" -- exact commands: `python scripts/mg.py verify --pack proof_library/bundles/<bundle>.zip` for each bundle
4. Print path to saved receipt file

**Critical implementation details:**
- Use `io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')` for Windows encoding (BUG 4 from CLAUDE.md)
- Use `Path.mkdir(parents=True, exist_ok=True)` for output dirs (BUG 5)
- Bundle paths in index.json use backslashes on Windows -- normalize with `Path()` constructor
- ZIPs may contain a top-level folder -- after extraction, find the directory containing `pack_manifest.json`
- Handle missing bundles gracefully (print warning, continue to next claim)
- No network calls anywhere -- temporal layer in existing bundles already has `beacon_status: "unavailable"` (DEMO-03)
- ANSI color codes for terminal output (same G/R/Y/C/B/DIM/X pattern as mg_client.py)
- Return exit code 0 if all verifications pass, 1 if any fail

**Create `demos/receipts/.gitkeep`** as empty file to ensure the directory exists in git.
