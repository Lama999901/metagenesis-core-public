**scripts/agent_coverage.py — Code Quality Coverage Analyst**

Use the standard boilerplate (encoding fix, REPO_ROOT, run() helper, colors) from agent_learn.py.

Main function `analyze()` does:
1. Run `python -m pytest tests/ --cov=backend --cov=scripts --cov-report=json --cov-report=term-missing -q --tb=no` via the `run()` helper. The JSON output goes to `coverage.json` in REPO_ROOT.
2. Parse `coverage.json` (it has structure: `{"files": {"path/to/file.py": {"summary": {"percent_covered": N, ...}, "missing_lines": [...], "executed_lines": [...]}}}`).
3. For each file in coverage data:
   - Read the actual source file
   - Find all `def function_name(` lines with their line numbers
   - Cross-reference with `missing_lines` to identify functions where ALL lines are in missing_lines (0% covered functions)
   - Also identify functions where >50% of lines are in missing_lines (low coverage)
4. Read `AGENT_TASKS.md` from REPO_ROOT (if it exists) and extract all PENDING task titles to avoid suggesting duplicates.
5. Build a report with sections:
   - "## Coverage Summary" — total files, average coverage %, files below 50%
   - "## Zero-Coverage Functions" — table: file | function | lines
   - "## Low-Coverage Functions (< 50%)" — table: file | function | coverage%
   - "## Suggested Tasks" — for each zero-coverage function, suggest a PENDING task (skip if similar title exists in AGENT_TASKS.md)
6. Write report to `reports/COVERAGE_REPORT_{YYYYMMDD}.md`
7. Print summary to stdout with colors (pass/fail counts)
8. Clean up `coverage.json` after parsing
9. Return exit code 0 if coverage > 60% overall, 1 otherwise

Add CLI: `python scripts/agent_coverage.py` (no args needed). Add `--summary` flag for one-line output.

**scripts/agent_evolve_self.py — Recursive Self-Improvement Analyst**

Use the same standard boilerplate.

Main function `analyze()` does:
1. Glob `reports/AGENT_REPORT_*.md` and `reports/WEEKLY_REPORT_*.md` — read all of them.
2. For each report, extract:
   - Date from filename
   - Task IDs mentioned (TASK-NNN pattern)
   - Key findings (lines starting with `- ` under `###` headers)
   - Any mentioned file paths
3. Load `.agent_memory/patterns.json` — extract known recurring patterns and their counts.
4. Analyze agent_research.py handlers:
   - For each `execute_task_NNN` function, count lines between its `def` and the next `def` (or EOF)
   - Flag handlers with fewer than 50 lines as "shallow" (may need enrichment)
   - Flag handlers with more than 200 lines as "complex" (may need splitting)
5. Cross-reference: Which patterns from patterns.json have NO corresponding handler in agent_research.py? These are "unaddressed patterns".
6. Identify report frequency: Are AGENT_REPORT files being generated regularly? Gap > 7 days = "stale research warning".
7. Build report with sections:
   - "## Agent Report History" — count of reports, date range, frequency
   - "## Handler Analysis" — table: handler | lines | verdict (shallow/ok/complex)
   - "## Recurring Patterns Without Handlers" — patterns from patterns.json not addressed
   - "## Recommendations" — specific actionable improvements
8. Write to `reports/SELF_IMPROVEMENT_{YYYYMMDD}.md`
9. Print summary to stdout with colors
10. Return exit code 0 always (advisory only)

Add CLI: `python scripts/agent_evolve_self.py` (no args). Add `--summary` flag.
