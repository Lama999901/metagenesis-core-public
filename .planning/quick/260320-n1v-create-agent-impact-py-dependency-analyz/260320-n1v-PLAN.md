**Function: parse_update_protocol()**
- Read UPDATE_PROTOCOL.md from REPO_ROOT
- Parse each change type section (### headings after "## CHECKLISTS BY CHANGE TYPE"):
  - "NEW CLAIM" -> extract all `[ ] path` lines as required files
  - "NEW TESTS" -> extract required file paths
  - "NEW VERIFICATION LAYER" -> extract required file paths
  - "NEW INNOVATION" -> extract required file paths
  - "NEW PUBLIC RELEASE" -> extract required file paths
  - "ARCHITECTURE CHANGE" -> extract required file paths
  - "PAYMENT METHOD CHANGE" -> extract required file paths
  - "SIGNIFICANT OUTREACH" -> extract required file paths
  - "REAL DATA MODE" -> extract required file paths
- For each change type, store a dict: { "trigger_patterns": [...], "required_files": [...] }
- Trigger patterns identify what file changes indicate this change type:
  - NEW CLAIM: "backend/progress/*.py" (new file, not runner.py)
  - NEW TESTS: "tests/**/*.py" (new or modified)
  - NEW LAYER: "scripts/mg_*.py" (new file)
  - NEW INNOVATION: "ppa/README_PPA.md" changes
  - NEW RELEASE: "system_manifest.json" version field change
  - ARCHITECTURE CHANGE: "docs/ARCHITECTURE.md" or "docs/PROTOCOL.md" changes
- Return dict of {change_type: {trigger_patterns, required_files}}

**Function: detect_change_type(changed_files: list[str]) -> list[str]**
- Given a list of changed file paths (relative to repo root), determine which change types apply
- Use the trigger patterns from parse_update_protocol()
- A change type matches if ANY trigger pattern matches ANY changed file
- Use fnmatch for glob patterns, exact match for specific files
- Return list of matching change type names

**Function: check_impact(changed_files: list[str] = None) -> dict**
- If changed_files is None, get files changed in HEAD commit: `git diff --name-only HEAD~1 HEAD`
- Call detect_change_type(changed_files)
- For each detected change type, check which required_files are NOT in changed_files
- Return: {"change_types": [...], "missing": {type: [files]}, "ok": bool}
- ok = True if no missing files (or no change types detected)

**Function: main()**
- Parse args: --summary (one-line output), --verbose (full detail), --files FILE... (explicit file list instead of git)
- Call check_impact()
- Print results:
  - If no change types detected: "No significant changes detected"
  - If all required files present: "Impact check PASS — all N downstream files updated"
  - If missing files: "Impact check: N missing downstream files for CHANGE_TYPE" + list each
- Exit 0 on pass, 1 on missing files (only in --strict mode, default is advisory)

- The trigger patterns should be HARDCODED (not parsed from UPDATE_PROTOCOL.md) because UPDATE_PROTOCOL.md doesn't contain machine-readable trigger info. Parse UPDATE_PROTOCOL.md only for the required_files lists.
- For parsing required files from UPDATE_PROTOCOL.md: look for lines matching `[ ] path` pattern inside each change type section (between ### headings). Extract the path part. Handle paths with arrows like "backend/progress/<claim_id>.py" — store as glob pattern "backend/progress/*.py".
- Include `if __name__ == "__main__": sys.exit(main())` at bottom.
- Add shebang: `#!/usr/bin/env python3`
- Add docstring explaining purpose, matching the style of other agent_*.py scripts.
