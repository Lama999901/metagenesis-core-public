FAQ

Skeptical? Good.

The questions we'd ask if we were you.

Does this guarantee my algorithm is correct?
No. MetaGenesis verifies that the evidence bundle contains what it claims and has not been modified. It does not verify the correctness of the underlying algorithm.
Can a sophisticated attacker fake a passing bundle?
MetaGenesis is tamper-evident, not a guarantee against all threat models. The semantic layer catches attacks that survive SHA-256 recomputation. Known limitations are documented in known_faults.yaml.
How is this different from just sending a Docker container?
A Docker container requires the verifier to run your environment and have compute available. A MetaGenesis bundle requires only Python and the verification script. No model access, no GPU, no network. Verification takes seconds.
Does the verifier need to trust MetaGenesis?
The verifier needs to trust the verification script (mg.py) and the protocol specification. Both are open source and auditable. Any third party can re-implement the verifier independently.
What does "patent pending" mean for open source users?
The code is MIT licensed — free to use, modify, and deploy. The provisional patent (USPTO #63/996,819) covers the protocol innovations. Open source use is unrestricted.
Does this replace MLflow or DVC?
No — MetaGenesis Core is complementary. MLflow tracks the experiment, MetaGenesis Core packages the result as a verifiable evidence artifact for external reviewers, auditors, and regulators.
